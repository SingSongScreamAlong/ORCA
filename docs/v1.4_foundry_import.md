# v1.4 — Foundry → ORCA Import (read-only against Foundry)

## Purpose

v1.1–v1.3 connected to Foundry and surfaced its data read-only (connector, admin endpoints,
preview UI). v1.4 takes the first step of *using* that data: an admin can **import Foundry
objects into ORCA as entities**. This is the smallest real write — and it writes only to
ORCA's own entity store, never to Foundry.

## What it does

`POST /api/v1/integrations/foundry/import` (admin-only) reads up to `limit` objects of one
Foundry object type (via the read-only connector) and materialises each as an **ORCA entity**:

| Request field | Meaning |
| ------------- | ------- |
| `object_type` | Foundry object type API name to read (e.g. `ExampleAircraft`, `OrcaEntity`). |
| `entity_type` | ORCA entity type for the batch — one of the fixed `EntityType` enum values. |
| `value_property` | Foundry property whose value becomes the entity `value` (e.g. `tailNum`). |
| `limit` | Max objects to read/import (1–50, default 10). |

The response reports `mode` (`mock`/`real`), `read`, `created`, `resolved_existing`,
`skipped`, and the resulting `entities`.

In the UI, the **Foundry** page (admin-only) gains an **"Import … into ORCA"** form when an
object type is selected: pick the value property and ORCA entity type, and import.

## Why entities (and why this is safe)

ORCA **entities** are the deduplicated, **non-asserting** reference primitive ("this phone
number / alias / vehicle exists") — created directly and deduped by `(entity_type, value)`,
exactly as ORCA's own entity service does. Importing reference entities therefore fits the
existing model and bypasses no review gate: the **assertions** (observations, relationships)
that *use* entities still pass through ORCA's review queue — *external/AI proposes, analysts
decide*.

Safety properties:

- **Read-only against Foundry.** The import only reads (the connector issues `GET`s); the only
  write is to ORCA's entity store.
- **Idempotent.** Entities dedupe by `(entity_type, value)`, so re-importing the same objects
  creates nothing new (`created: 0`, `resolved_existing: n`).
- **Admin-only.** Non-admins get a generic `403`; a denied import writes nothing.
- **Bounded & validated.** `limit` is capped at 50; `entity_type` must be a valid enum
  (invalid → `422`); objects missing the `value_property` are skipped, not guessed.
- **No secrets, no raw evidence.** Connector errors stay secret-free; only the chosen property
  value is imported — never raw evidence bytes.
- **Mode-labelled.** With Foundry disabled (dev/CI default) the deterministic mock supplies
  the objects (`mode: "mock"`); enable Foundry for live imports (`mode: "real"`).
- **Unverified by default.** Imported entities take the source object's `confidence` if
  present, else a modest default — they are leads to corroborate in ORCA, not findings.

## Example

```bash
# Import live ExampleAircraft tail numbers as 'vehicle' entities (mode: real with a token):
curl -s -X POST localhost:8000/api/v1/integrations/foundry/import \
  -H "X-ORCA-User: admin" -H "content-type: application/json" \
  -d '{"object_type":"ExampleAircraft","entity_type":"vehicle","value_property":"tailNum","limit":5}'
```

For a true round-trip, publish ORCA's `OrcaEntity` object type into Foundry (see
[`foundry_ontology_publish_guide.md`](foundry_ontology_publish_guide.md)), then import with
`object_type=OrcaEntity`, `value_property=value`.

## Tests

`tests/backend/test_foundry_import.py` (11 cases, mock mode, no network): creates entities
from imported objects, idempotency (re-import dedups), skips objects lacking the value
property, rejects invalid `entity_type` (`422`) and out-of-range `limit` (`422`), admin-only
`403`s across every non-admin role, and that a denied import persists nothing. Frontend
typecheck + build pass.

## Not in scope (yet)

Importing as observations/relationships (the reviewed assertion path), per-object
`entity_type` mapping, links, write-back to Foundry, and a scheduled/automated sync. Those
build on this admin, on-demand, read-only-against-Foundry import.
