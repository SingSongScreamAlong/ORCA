# Foundry Action Types

Reference for the ORCA → Foundry **action** mapping. Authoritative form:
`foundry/action_types.json` (generated by `python -m app.foundry_mapping.export` from
`backend/app/foundry_mapping/actions.py`). Each action mirrors an existing, audited ORCA
workflow and carries the same RBAC + case-role gate.

## Case-role gate tokens

- `""` — global capability only (not case-scoped at the action boundary).
- `member` — any active case member.
- `mutating` — case_manager / analyst / reviewer (roles that may create case material).
- `reviewer` — reviewer only (review decisions).
- `case_manager` — the case's manager (roster management, lifecycle).

Administrators satisfy every gate; the `mutating`/`reviewer`/`case_manager` checks also
require an **active** membership in the case.

`closeCase` / `archiveCase` map to a planned ORCA lifecycle endpoint; the action shape is
fixed here so the ontology is complete.

## Safety

No action is autonomous (`propose_only` is false for all human actions; assistive/AIP
output would be `propose_only` and route through the review queue). The following are
**forbidden workflows** and are never expressed as actions (asserted by tests): scraping,
dark-web collection, autonomous hunting, face search, offender/victim targeting, external
OSINT integration, Palantir live sync, Palantir production writes, AIP automation, bulk
social monitoring, direct contact.
