# Foundry Ontology Mapping — Objects & Links

Reference for the ORCA → Foundry **object** and **link** type mapping. The authoritative,
machine-readable form is `foundry/object_types.json` and `foundry/link_types.json`,
generated by `python -m app.foundry_mapping.export` from
`backend/app/foundry_mapping/`. This document is the human-readable companion. See
[`v0.9_palantir_foundry_mapping.md`](v0.9_palantir_foundry_mapping.md) for context.

### Property mapping kinds

Every property records its ORCA source field and a mapping kind:

- **direct** — a stored ORCA field (e.g. `OrcaEvidenceItem.sha256` ← `sha256`).
- **derived** — computed by ORCA (e.g. `OrcaEvidenceItem.verificationStatus`,
  `OrcaCase.handlingLevel`, `OrcaReportPackage.citedEvidenceCount`).
- **planned** — a target Foundry property not yet in the ORCA model (e.g.
  `OrcaCase.priority`, `OrcaCase.partnerAgency`, `OrcaSource.captureTime`).

Sensitive properties are flagged (`sensitive: true`) so partner exports and property-level
security can exclude them: e.g. `legalFlags`, `handlingNotes`, `storageUri`,
`OrcaObservation.text`, `OrcaEntity.value`, `OrcaReport.body`.

## Link types

| Foundry link type | Source → Target | Cardinality | ORCA basis |
| ----------------- | --------------- | ----------- | ---------- |
| `caseContainsSource` | OrcaCase → OrcaSource | ONE_TO_MANY | sources referenced by a case |
| `caseContainsObservation` | OrcaCase → OrcaObservation | ONE_TO_MANY | `Observation.case_id` |
| `caseContainsEvidenceItem` | OrcaCase → OrcaEvidenceItem | ONE_TO_MANY | `EvidenceItem.case_id` |
| `caseContainsEntity` | OrcaCase → OrcaEntity | MANY_TO_MANY | entities persist across cases |
| `caseContainsRelationship` | OrcaCase → OrcaRelationship | ONE_TO_MANY | `Relationship.case_id` |
| `caseContainsReport` | OrcaCase → OrcaReport | ONE_TO_MANY | `Report.case_id` |
| `caseContainsReportPackage` | OrcaCase → OrcaReportPackage | ONE_TO_MANY | `ReportPackage.case_id` |
| `caseHasCaseMembership` | OrcaCase → OrcaCaseMembership | ONE_TO_MANY | need-to-know roster |
| `sourceSupportsObservation` | OrcaSource → OrcaObservation | ONE_TO_MANY | `Observation.source_id` |
| `observationCitesEvidenceItem` | OrcaObservation → OrcaEvidenceItem | ONE_TO_MANY | `EvidenceItem.observation_id` |
| `observationSupportsRelationship` | OrcaObservation → OrcaRelationship | MANY_TO_MANY | **approved** supporting observations |
| `entityParticipatesInRelationship` | OrcaEntity → OrcaRelationship | MANY_TO_MANY | endpoints |
| `reportCitesObservation` | OrcaReport → OrcaObservation | MANY_TO_MANY | **approved** observations only |
| `reportPackageIncludesReport` | OrcaReportPackage → OrcaReport | ONE_TO_ONE | rendered report |
| `reportPackageIncludesEvidenceManifest` | OrcaReportPackage → OrcaEvidenceItem | MANY_TO_MANY | metadata + SHA-256 (no raw bytes) |
| `reviewDecisionAppliesToObservation` | OrcaReviewDecision → OrcaObservation | MANY_TO_ONE | decision subject |
| `reviewDecisionAppliesToEvidenceItem` | OrcaReviewDecision → OrcaEvidenceItem | MANY_TO_ONE | decision subject |
| `reviewDecisionAppliesToRelationship` | OrcaReviewDecision → OrcaRelationship | MANY_TO_ONE | decision subject |
| `auditEventRecordsActionOnObject` | OrcaAuditEvent → OrcaCase | MANY_TO_ONE | `target_type`/`target_id` |

No link grants a capability ORCA does not already have. Entity-to-entity traversal is via
`OrcaRelationship`, and only **approved** relationships are traversable (matching v0.5).
